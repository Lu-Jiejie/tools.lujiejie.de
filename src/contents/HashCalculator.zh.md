## 什么是哈希

[**哈希（Hash）**](https://zh.wikipedia.org/wiki/哈希函数)是一种将任意长度的数据转换为固定长度摘要值的过程。

相同输入始终会得到相同结果，而输入内容哪怕只修改一个字符，哈希值通常也会发生明显变化。

哈希广泛应用于文件校验、密码存储、数字签名、数据完整性验证以及各种安全场景。

## 如何使用

- 输入需要计算哈希值的文本内容。
- 选择一个或多个哈希算法。
- 工具会自动计算并显示对应的哈希结果。
- 复制生成的哈希值用于校验、比对或其他用途。

## 支持的算法

| 算法            | 适用场景                           |
| --------------- | ---------------------------------- |
| MD5             | 兼容旧系统、文件快速比对           |
| SHA-1           | 旧系统兼容，不建议用于新的安全场景 |
| SHA-256         | 最常见的安全哈希算法之一           |
| SHA-512         | 需要更高安全强度的场景             |
| SHA-3           | 新一代 SHA 系列算法                |
| BLAKE2 / BLAKE3 | 兼顾安全性与性能的现代哈希算法     |
| RIPEMD-160      | 部分协议和历史系统兼容需求         |
| SM3             | 中国商用密码标准哈希算法           |
| Whirlpool       | 较少使用的密码学哈希算法           |
| CRC32           | 数据校验与错误检测                 |

### 注意事项

- 哈希是单向运算，无法通过哈希值还原原始内容。
- 哈希与加密不同，加密数据可以解密，而哈希值无法反向恢复。
- MD5 和 SHA-1 已不适用于新的安全敏感场景。
- CRC32 主要用于检测数据传输或存储过程中的意外错误，不具备密码学安全性。
